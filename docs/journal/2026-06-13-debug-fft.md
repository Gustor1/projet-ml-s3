# 📅 2026-06-13 — Debug FFT & Re-run de l'Expérience 2

## 🎯 Objectif
Corriger le crash systématique de la méthode `spectral_subtraction` observé lors du premier run de l'Expérience 2.

## 🚨 Blocage identifié
Le script plantait sur 100% des fichiers avec l'erreur :  
`ValueError: operands could not be broadcast together with shapes`

## 🔍 Root Cause Analysis
Le bug se produisait lors de la reconstruction overlap-add dans la boucle FFT. La dernière frame de chaque signal audio était plus courte que la fenêtre FFT fixe (`nfft=2048`), provoquant un mismatch de forme lors de l'addition au tableau `result`.

## 🛠️ Solution appliquée
Ajout d'une gestion sécurisée des limites dans `experiments/compare_preprocessing.py` :
```python
# Avant (crash)
result[i:i+nfft] += clean_frame

# Après (sécurisé)
chunk_len = min(nfft, n - i)
result[i:i+chunk_len] += clean_frame[:chunk_len]

✅ Résultat

Re-run complet de l'expérience (180 inférences) pour garantir la cohérence des états de cache et des métriques.
Génération réussie de results/preprocessing_comparison.csv avec des valeurs WER/CER valides pour les 3 méthodes.

Documentation transparente de l'échec et du fix dans docs/experiment-2-final-comparison.md.
