# 📅 2026-06-13 — FFT Debug & Experiment 2 Re-run

## 🎯 Objective
Fix the systematic crash of the `spectral_subtraction` method observed during the first run of Experiment 2.

## 🚨 Issue Identified
The script crashed on 100% of the files with the following error:  
`ValueError: operands could not be broadcast together with shapes`

## 🔍 Root Cause Analysis
The bug occurred during the overlap-add reconstruction in the FFT loop. The last frame of each audio signal was shorter than the fixed FFT window (`nfft=2048`), causing a shape mismatch when adding to the `result` array.

## 🛠️ Applied Solution
Added safe boundary handling in `experiments/compare_preprocessing.py`:
```python
# Before (crash)
result[i:i+nfft] += clean_frame

# After (safe)
chunk_len = min(nfft, n - i)
result[i:i+chunk_len] += clean_frame[:chunk_len]
